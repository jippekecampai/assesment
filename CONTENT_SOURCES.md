# Campai Content Sources

Dit document registreert externe repositories die als inspiratie of bronmateriaal zijn gebruikt voor Campai Skills Academy, kandidaatvragen en assessmentscenario's.

## Gebruikslijn

- Content wordt omgezet naar Campai-eigen MSP-scenario's.
- Geen klantdata, credentials, IP-adressen, tenantnamen of klant-specifieke configuratie opnemen.
- Letterlijk hergebruik alleen wanneer de licentie dat toestaat en bronvermelding behouden blijft.
- Bij onduidelijke licentie: alleen gebruiken als inspiratie, niet als tekstuele bron.

## Bronnen

| Repository | Licentie-inschatting | Campai-gebruik |
|---|---|---|
| `Devinterview-io/azure-interview-questions` | Geen duidelijke licentie gezien tijdens review | Alleen inspiratie voor Azure concepten en interviewthema's |
| `Ditectrev/Microsoft-Azure-AZ-900-Microsoft-Azure-Fundamentals-Practice-Tests-Exams-Questions-Answers` | Licentie eerst verifiëren voor letterlijk gebruik | Inspiratie voor Azure Fundamentals en AZ-900 basislaag |
| `valakhosravi/Interview-Questions-Answers` | Unlicense op GitHub-pagina | Brede interviewstructuur en difficulty-indeling als inspiratie |
| `jondjones/ILoveTechTests` | Nog verifiëren voor letterlijk gebruik | Voorlopig niet letterlijk gebruikt |
| `malcolmmcdonald1982/M365-Assessment-Toolkit` | MIT op GitHub-pagina | M365 security assessment-thema's, attack-path prioritering en workload coverage |
| `UnitOneAI/SecuritySkills` | MIT op GitHub-pagina | Security review, attack-path, governance en secure automation rubrics |

## In de app verwerkt

- Nieuwe trainingsmodules voor Azure Fundamentals, Azure governance, M365 security assessment, Entra ID hardening, Teams/SharePoint secure collaboration, Intune baseline drift, attack-path triage, secure automation review en security awareness consulting.
- Nieuwe kandidaatvragen voor Azure, M365 security, SharePoint/Teams governance, attack-path prioritering, secure automation en consulting.
- Nieuwe conceptvragen in de vragenbank met bronlabels voor senior review.

## Follow-up

- Licenties van `Devinterview-io`, `Ditectrev` en `jondjones` expliciet vastleggen voordat tekst letterlijk wordt overgenomen.
- Bij productiegebruik bronvermelding meenemen in intern beheer of content-reviewproces.
- Vragen door senior engineers laten valideren voordat ze als definitieve assessmentvragen worden gebruikt.
